# Getting Started

## Quick Start

### 1. Start the Backend

```bash
git clone https://github.com/your-org/hono-analytics.git
cd hono-analytics
pnpm install

# Copy environment file
cp .env.example .env

# Start with Docker (PostgreSQL)
docker-compose up -d postgres

# Or use SQLite (no Docker needed)
export DATABASE_URL="sqlite:./packages/backend/analytics.db"

# Run migrations and start server
cd packages/backend
pnpm db:generate && pnpm db:migrate
cd ../..
pnpm --filter backend dev
```

### 2. Install SDK

```bash
npm install @hono-analytics/sdk
```

### 3. Add to Your App

```tsx path=null start=null
// Next.js - app/layout.tsx
import { NextJSAnalyticsProvider } from '@hono-analytics/sdk'

export default function RootLayout({ children }) {
  return (
    <NextJSAnalyticsProvider
      options={{
        apiKey: process.env.NEXT_PUBLIC_ANALYTICS_API_KEY!,
        projectId: process.env.NEXT_PUBLIC_ANALYTICS_PROJECT_ID!,
      }}
    >
      {children}
    </NextJSAnalyticsProvider>
  )
}
```

```tsx path=null start=null
// React - App.tsx
import { AnalyticsProvider } from '@hono-analytics/sdk/react'

function App({ children }) {
  return (
    <AnalyticsProvider
      apiKey="your-api-key"
      projectId="your-project-id"
    >
      {children}
    </AnalyticsProvider>
  )
}
```

```javascript path=null start=null
// Vanilla JS
import { initAnalytics, track } from '@hono-analytics/sdk'

const analytics = initAnalytics({
  apiKey: 'your-api-key',
  projectId: 'your-project-id'
})

track({ url: '/page-visit' })
```

### 4. Track Events

```tsx path=null start=null
import { useAnalytics } from '@hono-analytics/sdk'

function Button() {
  const analytics = useAnalytics()
  
  const handleClick = () => {
    analytics.track({ url: '/button/click' })
  }
  
  return <button onClick={handleClick}>Click me</button>
}
```

## Verify It's Working

Visit http://localhost:8000/health or run:

```bash
curl http://localhost:8000/health
```

Expected response:
```json
{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00.000Z"
}
```

### 2. Test Analytics Tracking

Visit the example app at http://localhost:3000 and interact with it. Check the browser console for analytics debug logs.

### 3. View Analytics Data

Check your analytics data:

```bash
curl -H "x-api-key: your-development-api-key" \
     "http://localhost:8000/metrics?range=7d"
```

## Troubleshooting

### Common Issues

**Database Connection Failed**
- Ensure PostgreSQL is running: `docker-compose ps`
- Check DATABASE_URL in `.env`
- Try SQLite fallback: `DATABASE_URL="sqlite:./packages/backend/analytics.db"`

**401 Unauthorized**
- Verify API key matches in backend `.env` and frontend config
- Check DEFAULT_API_KEY in backend environment

**CORS Errors**
- Ensure frontend origin is in CORS allowlist (see `packages/backend/src/server.ts`)
- Check that both frontend and backend are running

For more troubleshooting, see the [Troubleshooting Guide](/docs/troubleshooting).
