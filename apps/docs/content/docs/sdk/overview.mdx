# SDK Overview

The HONO Analytics SDK provides analytics tracking for web applications with TypeScript support and React integration.

## Installation

```bash
npm install @hono-analytics/sdk
```

## Quick Start

```tsx path=null start=null
// Next.js
import { NextJSAnalyticsProvider } from '@hono-analytics/sdk'

export default function RootLayout({ children }) {
  return (
    <NextJSAnalyticsProvider
      options={{
        apiKey: process.env.NEXT_PUBLIC_ANALYTICS_API_KEY!,
        projectId: process.env.NEXT_PUBLIC_ANALYTICS_PROJECT_ID!,
      }}
    >
      {children}
    </NextJSAnalyticsProvider>
  )
}
```

```tsx path=null start=null
// React
import { AnalyticsProvider } from '@hono-analytics/sdk/react'

function App({ children }) {
  return (
    <AnalyticsProvider
      apiKey="your-api-key"
      projectId="your-project-id"
    >
      {children}
    </AnalyticsProvider>
  )
}
```

```javascript path=null start=null
// Vanilla JS
import { initAnalytics, track } from '@hono-analytics/sdk'

const analytics = initAnalytics({
  apiKey: 'your-api-key',
  projectId: 'your-project-id'
})

track({ url: '/page-view' })
```

## Main Features

- **Automatic tracking**: Page views and route changes
- **User sessions**: 30-min timeout, device detection
- **TypeScript**: Full type safety
- **Lightweight**: ~8KB gzipped
- **Privacy-focused**: Self-hosted, GDPR-friendly
- **Framework support**: React, Next.js, vanilla JS

## API Methods

```typescript path=null start=null
// Track events
await analytics.track({ url: '/button-click' })

// Identify users
analytics.identify({ id: 'user-123', country: 'US' })

// Get session info
const sessionId = analytics.getSessionId()
const userId = analytics.getUserId()
```

## Configuration

```typescript path=null start=null
type TAnalyticsOptions = {
  apiKey: string              // Required: Your API key
  projectId: string           // Required: Project ID
  endpoint?: string           // Optional: API URL (default: localhost:8000)
  debug?: boolean             // Optional: Enable logging
  ignoreAnalytics?: boolean   // Optional: Disable all tracking
}
```
